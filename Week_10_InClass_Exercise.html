Stationarity Testing — ADF Test

Apply the Augmented Dickey-Fuller test before and after transformation.

🔬 2A — ADF Test on Raw Data

PYTHON 
def adf_test(series, name="Series"):
    result = adfuller(series.dropna())
    print(f"\n{'='*45}")
    print(f"  ADF Test — {name}")
    print(f"{'='*45}")
    print(f"  ADF Statistic : {result[0]:.4f}")
    print(f"  p-value       : {result[1]:.4f}")
    print(f"  Critical Values: 1%={result[4]['1%']:.3f}, 5%={result[4]['5%']:.3f}")
    if result[1] < 0.05:
        print("  ✅ STATIONARY  (p < 0.05 → Reject H₀)")
    else:
        print("  ❌ NON-STATIONARY  (p ≥ 0.05 → Fail to Reject H₀)")

adf_test(df["Passengers"], "Original Passengers")
🔍 Code Explanation
LineWhat it does
adfuller(series.dropna())Runs the ADF test. .dropna() removes any missing values first — the test will error if NaNs are present. Returns a tuple of results.
result[0]The ADF test statistic. More negative = stronger evidence against a unit root (i.e., stronger evidence of stationarity).
result[1]The p-value. This is the key number. Below 0.05 → stationary. Above 0.05 → non-stationary.
result[4]A dictionary of critical values at the 1%, 5%, and 10% significance levels. Your ADF statistic must be more negative than the critical value to reject H₀.
if result[1] < 0.05Decision rule: we reject the null hypothesis (H₀: series has a unit root) when p < 0.05, concluding the series IS stationary.

H₀ reminder: The null hypothesis of the ADF test is that the series is non-stationary (has a unit root). A high p-value (like 0.99) means we cannot reject that — the series is almost certainly non-stationary and needs transformation.

🔄 2B — Apply Transformations

PYTHON 
df["Log_Pass"]  = np.log(df["Passengers"])
df["Diff1"]     = df["Passengers"].diff()
df["Log_Diff1"] = df["Log_Pass"].diff()

adf_test(df["Log_Pass"],   "Log(Passengers)")
adf_test(df["Diff1"],      "First Difference")
adf_test(df["Log_Diff1"],  "Log + First Difference")
🔍 Code Explanation
TransformationWhat it does and why
np.log(df["Passengers"])Log transformation. Compresses large values and stabilises the variance — the seasonal swings that grow over time become roughly constant after logging. Fixes heteroscedasticity (changing variance).
df["Passengers"].diff()First-order differencing. Subtracts each value from the one before it: Y(t) − Y(t−1). This removes a linear trend by focusing on the change rather than the level. The first row becomes NaN (no previous value to subtract).
df["Log_Pass"].diff()Log + differencing combined. First stabilises variance, then removes trend. This is the standard pre-processing step before fitting ARIMA on data with both growing variance and a trend.

Why does this matter for ARIMA? ARIMA requires a stationary input. The d parameter in ARIMA(p,d,q) tells the model to apply first-order differencing automatically — so when you set d=1, ARIMA does the .diff() step internally.

📤 EXPECTED OUTPUT
ADF Test — Log(Passengers):       p-value = 0.4408  ❌ NON-STATIONARY
ADF Test — First Difference:      p-value = 0.0543  ❌ NON-STATIONARY (borderline)
ADF Test — Log + First Difference: p-value = 0.0421  ✅ STATIONARY
⚠️ Note

The raw data is not stationary. We need log transformation + differencing before fitting ARIMA. The d=1 parameter in ARIMA handles the differencing automatically — you don't need to pre-transform the data manually when using statsmodels.

🔍 Code Explanation
LineWhat it does
.rolling(window=3).mean()Slides a window of 3 periods along the series and takes the mean. Each output value is the average of the current and previous 2 observations. The first 2 rows are NaN (not enough history yet).
.rolling(window=12).mean()12-period SMA. Averages an entire year of monthly data — this smooths out all seasonal variation and reveals the pure trend. First 11 rows are NaN.
.ewm(alpha=0.3, adjust=False).mean()Exponential Weighted Moving Average. Weights older observations exponentially less. Alpha=0.3 gives 30% weight to the current observation, 21% to the previous, 15% to the one before, and so on. Low alpha = more smoothing.
.ewm(alpha=0.7, ...)Alpha=0.7 gives much more weight to recent data — the EWMA line hugs the original series more closely. High alpha = more reactive, less smoothing.
adjust=FalseUses the standard recursive EWMA formula. Setting adjust=True (the default) uses a slightly different weighted average formula — False is more consistent with how EWMA is defined in most textbooks.

🔮 3C — MA as a Naive Forecast

PYTHON 
df["MA_Forecast"] = df["SMA_12"].shift(1)

eval_df = df[["Passengers","MA_Forecast"]].dropna()
mae_ma  = mean_absolute_error(eval_df["Passengers"], eval_df["MA_Forecast"])

LineWhat it does

mean_absolute_error(actual, predicted)From sklearn. Computes the average of |actual − predicted|. Same units as the data (thousands of passengers).
np.sqrt(mean_squared_error(...))RMSE: square each error, take the mean, then square-root back to original units. Penalises large errors more than MAE.

🔍 Code Explanation
LineWhat it does
seasonal_decompose(..., model="additive")Assumes the series = Trend + Seasonal + Residual. Estimates the trend using a moving average, then estimates seasonal factors by subtracting the trend, then the residual is what's left.
period=12Tells the function the seasonal cycle length is 12 months (one year). Required for monthly data. Use period=4 for quarterly, period=7 for daily-with-weekly-seasonality.
result_add.plot()Returns a matplotlib figure with 4 subplots: original, trend, seasonal, and residual — each on its own row.
result_add.residAccesses the residual component of the decomposition — what's left after trend and seasonality are removed. Should look like random noise if the model is a good fit.
residuals_add.std()Standard deviation of the residuals — a smaller value means the decomposition explained more of the data variation.

Because seasonal variation grows as the trend increases, the multiplicative model fits better. Rule of thumb: if the seasonal peaks get taller as the series rises → multiplicative. If peaks stay roughly the same height → additive.

🔍 Code Explanation
LineWhat it does
np.log(train)Log-transforms the training data to stabilise variance. Done before differencing because the log makes the multiplicative seasonality additive.
train_log.diff().dropna()First-order differencing — removes the trend. Combined with the log, this produces a series that should be stationary and suitable for ACF/PACF analysis.
plot_acf(..., lags=40)Autocorrelation Function plot. Each bar shows the correlation between the series and its lag-k version. The shaded region is the 95% confidence band — spikes outside this are statistically significant.
plot_pacf(..., lags=40)Partial Autocorrelation Function plot. Shows the direct correlation at each lag, controlling for all shorter lags. A spike at lag 2 in the PACF means lag 2 predicts the series even after accounting for lags 1.

How to read these plots to choose ARIMA parameters:
• Count PACF spikes outside the shaded band → that's your p (AR order)
• Count ACF spikes outside the shaded band → that's your q (MA order)
• Spikes at lag 12 in both plots signal seasonal autocorrelation → you need SARIMA, not plain ARIMA

def evaluate(actual, predicted, model_name):
    mae  = mean_absolute_error(actual, predicted)
    rmse = np.sqrt(mean_squared_error(actual, predicted))
    mape = np.mean(np.abs((actual.values - predicted.values) / actual.values)) * 100
    r2   = r2_score(actual, predicted)
    return {"Model": model_name, "MAE": round(mae,2), "RMSE": round(rmse,2),
            "MAPE(%)": round(mape,2), "R²": round(r2,4)}

results = [
    evaluate(test, ses_fc,    "SES (α=0.3)"),
    evaluate(test, des_fc,    "Holt's DES"),
    evaluate(test, hw_fc,     "Holt-Winters"),
    evaluate(test, arima_fc,  "ARIMA(2,1,1)"),
    evaluate(test, sarima_fc, "SARIMA(1,1,1)(1,1,1,12)"),
]

results_df = pd.DataFrame(results).set_index("Model")
results_df["Rank"] = results_df["MAPE(%)"].rank().astype(int)
results_df = results_df.sort_values("MAPE(%)")
print("\n📊 Model Comparison (sorted by MAPE):")
print(results_df.to_string())
